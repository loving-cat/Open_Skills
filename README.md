# 专家团模式 (Expert Team Mode)

> Claude Code Skill - 通过"规划→任务分解→专家执行→疑难调度"流程解决复杂开发问题

[![Claude Code](https://img.shields.io/badge/Claude_Code-Skill-blue)](https://github.com/anthropics/claude-code)

## 简介

专家团模式是一个 Claude Code Skill，通过模拟真实团队协作流程来处理复杂开发任务：

- **架构师** - 需求分析、技术选型、任务分解
- **前端专家** - UI组件、页面开发、样式实现
- **后端专家** - API开发、业务逻辑、服务实现
- **数据库专家** - 数据模型、迁移脚本、查询优化
- **DevOps专家** - Docker配置、CI/CD、部署脚本
- **安全/性能/测试专家** - 疑难攻坚增援

## 快速配置

### 方法一：自动配置（推荐）

运行以下命令：

```bash
# Windows (PowerShell)
mkdir -p "$env:USERPROFILE\.claude\skills\Think"
cp expert-team-mode.md "$env:USERPROFILE\.claude\skills\Think\SKILL.md"

# macOS / Linux
mkdir -p ~/.claude/skills/Think
cp expert-team-mode.md ~/.claude/skills/Think/SKILL.md
```

### 方法二：手动配置

1. 创建目录：

   ```
   ~/.claude/skills/Think/
   ```

2. 将 `expert-team-mode.md` 复制到该目录，并重命名为 `SKILL.md`

3. 最终路径：

   ```
   ~/.claude/skills/Think/SKILL.md
   ```

### 验证配置

重启 Claude Code 后，输入 `/Think` 或 `/专家团`，如果看到专家团模式的响应，说明配置成功。

## 工作流程

```
┌─────────────────────────────────────────┐
│  Phase 1: 架构师规划                      │
│  - 需求分析 → 技术选型 → 任务分解          │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  Phase 2: 任务分配                        │
│  Task1(前端) Task2(后端) Task3(数据库)    │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  Phase 3: 并行执行                        │
│  无依赖任务同时启动，有依赖任务串行执行      │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  Phase 4: 疑难调度（可选）                 │
│  遇到困难自动增派专家协作攻坚               │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  Phase 5: 验收                            │
│  测试专家验证功能，输出报告                 │
└─────────────────────────────────────────┘
```

## 示例

**输入：**

```
/专家团 创建一个博客系统，包含文章管理、评论、用户认证功能
```

**执行过程：**

1. 架构师输出技术选型方案和任务分解
2. DevOps专家搭建项目基础架构
3. 数据库专家设计表结构
4. 后端专家开发 API
5. 前端专家实现页面
6. 测试专家验收

```
~/.claude/skills/

```

## 注意事项

1. **先规划后执行** - 必须先输出 Plan 和任务分解，再分配执行
2. **真正的并行** - 无依赖任务会同时启动多个 Agent 并行执行
3. **疑难调度** - 执行困难时自动增派专家协助
4. **资源节约** - 只调度必要的专家
